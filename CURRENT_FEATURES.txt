📊 9. DATABASE & MODELS
═══════════════════════════════════════════════════════════════════════

💾 SQLite Database
──────────────────────────────────────────────────────────────────────
✅ Lightweight and portable
✅ No separate server needed
✅ Easy backup
✅ Perfect for development


📋 Data Models
──────────────────────────────────────────────────────────────────────
✅ User (Django built-in)
✅ Patient (linked to User)
✅ Doctor (with user link - NEW!)
✅ Appointment
✅ Payment
✅ TimeSlot


🔗 Relationships
──────────────────────────────────────────────────────────────────────
✅ One-to-One: User → Patient, User → Doctor
✅ One-to-Many: Doctor → Appointments
✅ One-to-Many: Patient → Appointments
✅ One-to-Many: Appointment → Payments
✅ One-to-Many: Doctor → TimeSlots


📝 Model Fields
──────────────────────────────────────────────────────────────────────
Patient:
  - User link, phone, address
  - Date of birth, gender, blood group
  - Medical history, allergies
  - Timestamps

Doctor:
  - User link (NEW!)
  - Name, specialization, email, phone
  - Qualifications, experience, bio (NEW!)
  - Consultation fee, availability
  - Timestamps

Appointment:
  - Patient, Doctor links
  - Date, time
  - Reason, symptoms
  - Status, payment status
  - Zoom meeting details
  - Amount
  - Timestamps

Payment:
  - Appointment link
  - Amount, method
  - Transaction ID
  - bKash transaction ID
  - Invoice number
  - Status, payment date
  - Timestamps

TimeSlot:
  - Doctor link
  - Weekday, start time, end time
  - Availability status
